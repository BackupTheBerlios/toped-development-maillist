> Hi,
> Not a FreeBSD user, but...
> Obviously some of the autotools subscripts I'm using need attention or
> replacement. It looks like the configure expects the headers to be in
> /usr/include/...
>
> You can help the script adding an include directory at the command line
> - something like this:
>  > configure <...options...> CPPFLAGS=-I/usr/local/include
> If you have troubles with include directories, you might have similar
> troubles with library directories. If that's the case you could need to
> point-out the location of the libraries for the linker (LDFLAGS). Should
> be pretty similar - try "configure --help" for more details.
>
> It seems also that some of the options are redundant: g77 and also
> --with-* stuff. The latter is probably more appropriate for wx
> compilation. As I said though - I'm not using FreeBSD , so whatever
> works for you.
